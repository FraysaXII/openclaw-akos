# Agent Instructions For I67

## Binding Mandates

1. **Research before roadmap.** Do not turn this scaffold into a phase plan until internal and external research has been done.
2. **Challenge the scaffold.** RevOps may be the wrong name. The 6 x 3 matrix may be the wrong public structure. Test both.
3. **Surface decisions.** If a choice affects channels, pricing, pages, tooling, CRM, partner economics, or KPI definitions, write it as a decision point and ask the operator.
4. **Inherit I66.** Do not break branded-house architecture, baseline-reality register rules, or the P6 template registry.
5. **No tooling-first work.** Do not select HubSpot, Attio, Airtable, Notion, PostHog, or a custom dashboard before workflow needs are explicit.
6. **No public promotion by accident.** `/services` and `/how-we-work` remain direct-access until an I67 decision promotes them.
7. **Use transcripts carefully.** Mine them for funnel signals and objection patterns; redact personal and company-identifying details.
8. **Produce evidence.** Every recommendation needs source notes and rejected alternatives.

## First Agent Task

Create a research inventory:

- internal sources read,
- external sources to fetch,
- operator interview questions,
- unknowns,
- decision points likely to block implementation.

Then stop for operator review before drafting phases.
