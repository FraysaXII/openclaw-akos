# User journeys — AKOS Planning Workspace Panel

> 5 journeys, each with `trigger → glance → action → outcome` and a grasp test that defines what "intuitive" looks like. These journeys gate the I65 build.

## J-65-1 — Founder catch-up scan (<30 seconds)

```mermaid
flowchart LR
  trigger["18:30 · Founder opens /operator/planning/ at end of day"]
  glance["Hero: '47 active · 5 reports today · 3 not reviewed in 21+ days' · search bar empty · filters at default (all statuses)"]
  action["Eye scans the 5 most-recent cards (sorted by last_review_at DESC) · clicks one whose excerpt mentions an unfamiliar phase"]
  outcome["Lands on drill-in · skims the new report · closes laptop"]
  trigger --> glance --> action --> outcome
```

**Grasp test passes only if** the hero band carries the count of "reports today". Without it, the founder scans cards to count — which is a 90-second job, not a 30-second one. The 5 most-recent cards must be visible above the fold on a 1440×900 viewport.

## J-65-2 — Operator drilling into "my" initiative (<90 seconds)

```mermaid
flowchart LR
  trigger["Operator (DevOPS) gets a Slack ping: 'I63 P8 needs your sign-off' · clicks #governance link"]
  glance["Lands on /operator/planning/?role=devops · 3 cards visible · I63 card pulses ember (stale chip · last review 4 days ago)"]
  action["Click I63 card · drill-in loads · 'Next: P8 process-list harmonisation' callout visible · click 'Open in Cursor' chip"]
  outcome["Cursor opens master-roadmap.md at the right line · operator writes the sign-off · saves · drill-in auto-revalidates after 60s"]
  trigger --> glance --> action --> outcome
```

**Grasp test passes only if** the role filter persists across page loads, the "Next phase" callout is the dominant signal in the drill-in, and the Cursor deeplink works from the chip without manual path-pasting.

## J-65-3 — Authoring the next report inline (<180 seconds)

```mermaid
flowchart LR
  trigger["End of phase · need to add a new report under /reports"]
  glance["On drill-in · reports gallery has 'Add report' chip · clicking opens a small dropdown of common report_kinds (impeccable-shape, journeys, uat, ...)"]
  action["Pick 'uat' · the chip generates a `cursor://file/...` link to a pre-filled new file at reports/uat-i65-2026-05-XX.md · Cursor opens with frontmatter prefilled · operator writes UAT · saves"]
  outcome["Within 60s the new report appears in the gallery · stream picks it up next refresh"]
  trigger --> glance --> action --> outcome
```

**Grasp test passes only if** the operator never types a path manually. The chip generates the path; Cursor handles the rest. The pre-filled frontmatter matches the AKOS report-kind conventions exactly.

## J-65-4 — Auditor reviewing closed initiatives (<120 seconds)

```mermaid
flowchart LR
  trigger["Auditor opens showcase.holistikaresearch.com/showcase · clicks 'Planning workspace tour' fixture link"]
  glance["Demo mode loads · banner top-right · hero: '12 active · 8 closed · 2 archived' · filter chips include 'closed'"]
  action["Click 'closed' filter · 8 cards visible · click any · drill-in shows the closure decision prominently in the cross-link rail"]
  outcome["Auditor reads the closure decision (one paragraph) · screenshot · close tab"]
  trigger --> glance --> action --> outcome
```

**Grasp test passes only if** demo mode shows fixture data only (no real workspace text), the closure decision is the **first** item in the cross-link rail for closed initiatives (not buried), and the closure decision rendering is one paragraph (not a table dump).

## J-65-5 — Time-travel post-mortem (<240 seconds)

```mermaid
flowchart LR
  trigger["Cycle 03 closed last week · operator wants to know 'what was active 14 days ago when we made D-IH-58-3'"]
  glance["From any drill-in · click 'View at date...' chip in cross-link rail · date picker shows last 30 days · pick 2026-04-23"]
  action["URL gains ?ref=7a3b1c2 · banner appears at top: 'Viewing at 2026-04-23 · back to live →' · workspace re-renders with content from that SHA"]
  outcome["Operator reads the master-roadmap as it was · understands the original intent · returns to live with one click"]
  trigger --> glance --> action --> outcome
```

**Grasp test passes only if** the time-travel banner is loud enough that no one mistakes historical for live (per R-65-4), and the date→SHA resolver returns reliably for the most recent merged commit on `main` at the picked date.

## Grasp tests, summary

For each journey, what "intuitive" means:

1. **J-65-1**: hero carries the count. Cards are scannable by status pill. No filtering required.
2. **J-65-2**: role filter persists. "Next phase" callout dominates drill-in. Cursor deeplink works.
3. **J-65-3**: no manual path typing. Frontmatter pre-filled. Gallery refreshes within 60s.
4. **J-65-4**: closure decision dominates closed-initiative drill-ins. Showcase shows fixtures only.
5. **J-65-5**: banner loud, "back to live" obvious, date→SHA reliable.

## Out of journey scope

- **Editing master-roadmap content.** Cursor handles authoring. The panel is read+navigate only.
- **Cross-AKOS-instance views.** Single AKOS instance for now.
- **Initiative dependency graph.** Defers to P7 if requested.
