## 1. Context

The fourth v3.1 methodology versioning event of the day. Pattern continues from D-IH-70-Z + D-IH-70-AA + D-IH-70-AB:

> Operator surfaces v2.7-prose-only concept → agent runs evidence sweep → AskQuestion ratifies inline → schema extension + concrete rows + canonical doctrine doc + decision register + checkpoint.

The novelty in P8.5: this is the first event where the operator-overrode the agent's recommendation **and** simultaneously surfaced a **new schema dimension** during the override deliberation. The override happened on Q5 (`competitor_intelligence_target`) where the recommended action was "reject as audit-subject not relationship". The operator's response: *"we are researching having GOI/POI competitors too will help our intel ops across the board"* — which raised a follow-on question: should there be a column for "stance" (their stance toward us, our stance toward them) so IntelligenceOps queries can score relationship posture?

The answer (operator-ratified after agent recommendation): **yes**, single-column `stance` on GOI_POI_REGISTER.csv with enum `ally/neutral/enemy/unknown`, codifying the v2.7 doctrine quoted in `GOI_POI_STANCE_DOCTRINE.md` §1.
