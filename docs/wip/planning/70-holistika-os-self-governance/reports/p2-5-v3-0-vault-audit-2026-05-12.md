# I70 P2.5 — Full v3.0 vault audit (2026-05-12)

> Resolves operator-ratified `Question p3-d-ih-70-b: option B and do a regression over the file structure, see if it really suits us or need rework`. Catalogs every directory under `docs/references/hlk/v3.0/` with verdict, area-role ownership, and target migration wave for Phase 4.5. **Inline-ratify gate** at §6 below — operator spot-checks 10-15 verdict rows before Phase 4.5 wave 1 starts (per H1 ratification + `akos-inline-ratification.mdc`).

## 1. Audit method

1. Walked the `docs/references/hlk/v3.0/` tree via `os.walk` (173 directories total).
2. Cross-referenced each directory against:
   - [`baseline_organisation.csv`](../../../../references/hlk/compliance/baseline_organisation.csv) (area + role ownership).
   - [`process_list.csv`](../../../../references/hlk/compliance/process_list.csv) (functional ownership).
   - [`INITIATIVE_REGISTRY.csv`](../../../../references/hlk/compliance/INITIATIVE_REGISTRY.csv) (active initiative folder?).
   - I70 plan [`Phase 3 ratifications`](../../../../../.cursor/plans/holistika_os_self-governance_foundation_63841b81.plan.md) (D-IH-70-A through K + 12 conundrums).
3. Verdict assigned per the schema declared in plan §2.5:
   - **`keep`** — actively used, properly owned, federated home or already correct.
   - **`merge`** — duplicate or overlap; consolidate with another folder (Envoy Tech Lab × 2 is the canonical example).
   - **`move`** — misplaced ownership; relocate to correct area-role (most compliance authoring artifacts; Marketing M3 redesign moves).
   - **`populate`** — empty-but-owned folder that should have content (charters per P5 + P4.7).
   - **`delete`** — empty + unowned + zero cross-link (rare; mostly applied at P9 §9.8 to `temp-move-or-delete-*`).
   - **`rename`** — accurate but poorly named.
4. Migration wave assigned per Phase 4.5 §4.5.5: Wave 1 (Brand pattern test), Wave 2 (compliance master + dimensions), Wave 3 (all remaining areas + legacy-link sweep + delete).
