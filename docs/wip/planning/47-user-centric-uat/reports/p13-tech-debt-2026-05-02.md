# I47 P13 — Tech debt cleanup (4 items closed)

## Item 1: `sync_hlk_neo4j` 6-dimension write extension (closes I46 drift canary catch)

The I46 P2 drift canary surfaced that `scripts/sync_hlk_neo4j.py:sync_csv_graph()` only wrote 4 of 10 HLK dimensions (Role / Process / Program / Topic). The 6 axis-6 dimensions (Persona / Channel / Sourcing / Skill / TouchpointKitCell / Policy) were projected by `akos.hlk_graph_model.build_holistik_ops_axis_graph()` but never sent to Neo4j.

### Fix

Extended `akos/hlk_neo4j.py:sync_csv_graph()`:
- Added `build_holistik_ops_axis_graph` to imports (alongside existing program/topic builders)
- 6 new MERGE batches (one per axis-6 label) keyed on each dimension's primary key (`persona_id`, `channel_id`, `vendor_id`, `skill_id`, `cell_id`, `policy_id`)
- New per-axis UNDER_TOPIC edge writer using a label→key dispatcher dict
- Return value extended with `personas_written` / `channels_written` / `sourcing_written` / `skills_written` / `cells_written` / `policies_written` per-dim counts

### Impact

After P15 closure live sync, `scripts/graphrag_drift_canary.py` should report `csv=N neo4j=N [OK]` for all 10 dimensions instead of the current `csv=N neo4j=0 [DRIFT]` for the 6 axis-6 dimensions. This unlocks I46 P5 conditional GraphRAG ship which depends on persona/channel/skill/policy nodes existing in the live KG.

## Item 2: `sync_compliance_mirrors_from_csv` boolean column emit fix

I46 P7 operator-applied step failed with `22P02: invalid input syntax for type boolean: ""` because the script emitted empty-string `''` for `tools_required_waived` when the CSV cell was blank.

### Fix

Extended `_emit_skill_registry_upserts()` with explicit boolean handling:
- New constant `BOOL_COLUMNS = {"tools_required_waived"}`
- For each row, boolean columns receive `true` / `false` / `NULL` keywords (NOT quoted text)
- Recognises `true|1|yes|y` / `false|0|no|n` (case-insensitive) and falls back to `NULL` for empty / unknown values
- All other (TEXT) columns continue using the existing `_sql_text_literal()` quoting path

### Validation

`tests/test_i47_p13_tech_debt.py::test_skill_registry_emit_handles_boolean_columns` exercises 4 input cases (`true`, `false`, empty, `TRUE`) and asserts:
- `true` / `false` keywords appear in emitted SQL
- `NULL` keyword for empty boolean values
- TEXT columns still use `''` literals (no regression)

## Item 4: `compliance.eval_run` live writes wiring

The I45 P4 migration created `compliance.eval_run` table but no code wrote to it. P10 added the schema extensions (persona_id / difficulty_class / scenario_class / judge_scores). P13 item 4 wires the live writes.

### Delivered

`akos/eval_harness/eval_run_writer.py` (NEW):
- Best-effort: silently no-ops when `SUPABASE_URL` / `SUPABASE_SERVICE_ROLE_KEY` env vars are missing
- Uses stdlib `urllib.request` (no new dependency)
- Posts to `/rest/v1/eval_run` with `Content-Profile: compliance` header (PostgREST schema-level dispatch)
- HTTP errors logged at WARNING; never raise (eval is the primary surface; observability is advisory)

`scripts/eval.py` integration:
- Calls `write_scorecard_rows(sc, source_git_sha=git_sha)` after every `run_modes()` returns
- Records writer stats in `Scorecard.metadata['eval_run_writer']` for visibility
- Defensive try/except wrapping (any writer crash falls through to `metadata['eval_run_writer_error']`)

### Activation

The writer activates when `SUPABASE_URL` AND `SUPABASE_SERVICE_ROLE_KEY` env vars are both set (standard service-role posture from I26-vintage). For Tier B GitHub Action runs, the secrets are already operator-managed (I45 P6).
