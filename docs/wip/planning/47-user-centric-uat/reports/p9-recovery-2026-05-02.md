### `scripts/recovery_chaos_runner.py` (NEW)

The single OPT-IN real-chaos surface per D-IH-47-L. 4 safety gates verified BEFORE any destructive op:

1. `AKOS_REAL_CHAOS_OK=1` env var (default refuse; exit code 10)
2. `NEO4J_URI` host MUST NOT match the MasterData host (default forbidden list + extensible via `AKOS_REAL_CHAOS_FORBIDDEN_HOSTS`; exit code 11)
3. Operator interactive confirmation (must type the URI host) OR `--non-interactive --confirm-test-host=<host>` matching the URI (exit code 12)
4. Live mode requires `NEO4J_AURA_API_KEY` (exit code 14); without it, exits non-zero with actionable error rather than half-rotating

Live rotation path is itself PLANNED-ONLY until operator approves first chaos run and pins the Aura API contract — this prevents an in-flight operator-mistake scenario.

Chaos report is ALWAYS written to `artifacts/chaos/real-chaos-neo4j-rotation-<UTC-timestamp>.json` with full gate-check audit trail (which gates passed, which refused, why).

## Coverage discipline highlights

### The "real-vs-mocked-API divergence" gap (D-IH-47-L origin)

The I46 password-truncation lesson (D-IH-32-Q): the synthetic mock said HTTP 401, the real Aura also said HTTP 401, but the operator's actual-vs-intended-password gap was invisible to the mock because the mock didn't go through the full credential resolution path.

The real-chaos runner closes this gap:
- Synthetic recovery scenarios SCN-REC-AUT-001/002-V1 catch the surface failure mode (auth_error)
- Real-chaos SCN-REC-CHA-001-V1 catches the deeper "real Aura HTTP API behaves the same as the mock under known conditions" — but does so SAFELY (4 gates)
- Test scenarios SCN-REC-AUT-001/002-V1 also test that the response correctly references the I46 D-IH-32-Q diagnosis (the password-truncation context becomes part of the debugging surface)

### Cost breach discipline (correct outcome = REFUSE)

3 cost-breach scenarios (SCN-REC-CST-001/002/003-V1) ALL expect REFUSE:
- The system MUST NOT continue spending after a cost ceiling is breached
- This is distinct from ESCALATE (which would mean "ask operator"; cost-breach is "stop NOW")
- POL-COST-* policies set the ceilings; the runtime must enforce

### Real-chaos is intentionally `impossible` (in the test sense)

SCN-REC-CHA-001-V1 has `difficulty_class=impossible` because:
- Without `AKOS_REAL_CHAOS_OK=1`: scenario "fails" (REFUSED) at gate 1 — correct outcome
- With env set + matching gates: scenario passes (PLANNED report)
- Without env set: cassette replay path is not even taken

This means the scenario is "answerable" only via correct refusal — the same pattern as our other `impossible` scenarios (weather, future revenue forecast).
