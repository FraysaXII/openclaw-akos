## 5. Strictness ladder (C-71-1 closure)

C-71-1 (Pack A1 strictness ladder) was open through Round 1 planning with default `soft-for-30-days-then-strict`. Operator override at plan finalization 2026-05-14 ratified **strict-day-1** (D-IH-71-F). Rationale: brand voice failures on customer-visible artifacts (per the SUEZ deck precedent) are not recoverable by a 30-day grace window — every PR opened during the grace period would carry the LLM-tone drift forward. Per-rule allow-listing via `register-pack.yml` provides the granular relief that a global soft-mode would have provided; `AKOS_BRAND_VOICE_REGISTER_SOFT=1` env stays as an emergency triage escape hatch (CI never sets it; operators may set it locally for one-off bypasses with documented justification).
