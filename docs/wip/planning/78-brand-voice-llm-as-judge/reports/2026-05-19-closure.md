### §2.1 Closure shape — P1+P2-only with P3-P5 forward-chartered

**Executive call:** Close I78 today at engineering-done (P1+P2 shipped); forward-charter Strand C (bias audit) + Strand D (promotion-to-strict) to a successor follow-up initiative.

**Rationale:** Three convergent reasons.
1. The [I86 cluster burndown plan §6 axis-2 pragmatic-closure executive call](../../86-initiative-cluster-execution-coordinator/cluster-burndown-plan.md) explicitly allows this shape ("forward-charter any closure UAT to a successor initiative if operator-cadence permits at the time"). The cluster's Wave H budget is "I78 closure ratify carries optional `D-IH-78-CLOSURE` mint if operator chooses to formally close P1-only".
2. Strand D ratification is gated on Strand C bias-audit data — running the audit requires a live LLM provider configured under `config/openclaw.json.example` AND a 30-day window AND operator capacity to review daily bias samples. None of those are available in the subagent's execution context.
3. Closing on engineering-done preserves operator intent (I78 was named for a reason; the chassis ships) without speculative-promotion debt on bias-audit rows that have not been measured yet.

**Reversibility:** Single-diff at three sites — (a) flip [`INITIATIVE_REGISTRY.csv`](../../../references/hlk/v3.0/Admin/O5-1/People/Compliance/canonicals/INITIATIVE_REGISTRY.csv) `status: closed → active` on the `INIT-OPENCLAW_AKOS-78` row; (b) mint a new decision row `D-IH-78-A2` (or `D-IH-78-REOPEN`) in [`DECISION_REGISTER.csv`](../../../references/hlk/v3.0/Admin/O5-1/People/Compliance/canonicals/DECISION_REGISTER.csv) citing the re-open trigger; (c) flip [`master-roadmap.md`](master-roadmap.md) frontmatter `status: active`. A successor strict-mode-promotion initiative (e.g. `INIT-OPENCLAW_AKOS-78-PROMOTE` or a new initiative number) is an alternative: forward-charter via candidate stub under `_candidates/`.

**Operator override mechanism:** Reply in the next chat turn with one of: (a) "reopen I78 — strand C live now" → revert flips above and mint `D-IH-78-PROMOTE` prep work; (b) "spin off successor initiative" → I'll author the candidate stub; (c) accept and continue — no action needed (default if silent for 14 days per [`akos-agent-checkpoint-discipline.mdc`](../../../../.cursor/rules/akos-agent-checkpoint-discipline.mdc) soft-pause auto-clear).

### §2.2 Provider scope — mock-only at P1+P2; live providers forward-chartered

**Executive call:** Ship only the `mock` provider with deterministic in-process verdicts in P1. The live provider dispatchers (`anthropic` / `openai` / `deepseek-r1-local` / `ollama`) raise `NotImplementedError` with a pointer to the strict-mode-promotion follow-up.

**Rationale:** Live provider wiring requires real API keys (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, RunPod URL+token, Ollama endpoint) that the subagent's execution context lacks. Wiring without keys creates broken paths that fail silently. The `mock` provider gives a complete chassis exercise (Pydantic models load, CLI dispatches, cache key is deterministic, release-gate INFO row passes) without the hidden-failure surface. Live wiring lands together with the bias-audit cadence in the same follow-up initiative — they need each other to validate (bias tests measure a real provider's behavior, not a mock).

**Reversibility:** Drop-in replacement at [`scripts/judge_brand_voice.py::_live_provider_call`](../../../../scripts/judge_brand_voice.py). The chassis (`akos.brand_voice_judge.JudgeProvider` Literal enum + `JudgeRequest`/`JudgeVerdict` Pydantic models) already enumerates the supported providers; concrete `if req.provider == "anthropic": ...` branches plug in without changing the chassis. The cost-tracking surface (`JudgeVerdict.cost_eur`) and cache-key contract (`JudgeRequest.cache_key`) are already shipped.

**Operator override mechanism:** When ready to wire a live provider, author the dispatch branches in `_live_provider_call`, add the API key bindings to `config/environments/*.env.example`, populate `config/openclaw.json` accordingly, and re-run `py scripts/judge_brand_voice.py --provider <name> --prose "..."`. No chassis changes needed.

### §2.3 Release-gate row category — INFO advisory; not PASS/FAIL

**Executive call:** The release-gate `run_brand_voice_judge_self_test` row category is `INFO`, not `PASS`/`FAIL`. The self-test exercises the chassis but does not gate CI on real prose verdicts.

**Rationale:** Pattern parity with three contemporaneous advisory-first validators — `validate_initiative_program_anchors.py` (I86 P1 stayed INFO until Stage B FK promotion), `validate_external_render_trail.py` (I86 Wave E launched INFO; promoted to FAIL only at Wave F D-IH-86-Q after the render-pending tracker reached zero), and `validate_locale_orthography.py` (Wave F launched INFO; EN promoted at Wave G B-G1 D-IH-86-R). Promoting the judge to PASS/FAIL today would create CI block on every commit while the bias-audit data is missing — exactly the failure mode Strand C exists to prevent.

**Reversibility:** Single-line edit in [`scripts/release-gate.py::run_brand_voice_judge_self_test`](../../../../scripts/release-gate.py) — change `results.append(("INFO", ...))` to `results.append(("PASS" if judge_ok else "FAIL", ...))`. This is the same shape as the orthography promotion pattern (commit reference: see CHANGELOG `D-IH-86-R` Wave G B-G1 entry).

**Operator override mechanism:** When Strand C audit data exists + Strand D D-IH-78-PROMOTE ratifies, the strict-mode-promotion follow-up initiative flips the INFO category to PASS/FAIL in its phase-2 closure commit.

## §3 Deliverables landed at this closure

### §3.1 Pydantic chassis — [`akos/brand_voice_judge.py`](../../../../akos/brand_voice_judge.py)

7 Pydantic models per [`CONTRIBUTING.md`](../../../../CONTRIBUTING.md) §"Python Code Standards":

| Model | Purpose |
|:---|:---|
| `JudgeRequest` | One judge invocation: prose under audit + audience class + provider + prompt version + locale + mode + optional surface path + optional canonical-set override. `cache_key()` method emits SHA256 over prose + audience + provider + prompt-version + locale + sorted canonical-paths (surface-path intentionally excluded for cache reuse across surfaces). |
| `JudgeVerdict` | Structured judge response: verdict (pass/fail/info) + severity + rationale + suggested rewrite (None when verdict=pass; required when fail/info) + canonicals cited + provider used + prompt version + cost + latency. Cross-field validators enforce (pass→no rewrite) + (fail→severity not pass). |
| `JudgePromptVersion` | Versioned judge-prompt registry entry — semver pattern, audience-class binary, category enum, canonical basis tuple, template sha256, mint date. Bumping invalidates cached verdicts per C-78-6. |
| `JudgeBiasMitigation` | Strand C bias-test result row — category (position/verbosity/self-preference/authority) + test name + sample size + FP rate [0,1] + FN rate [0,1] + measured-at + measured-by + notes. Schema-locked at P1 so the bias-audit log doesn't drift when Strand C activates. |
| `BrandVoiceJudgeConfig` | Operator-tunable knobs loaded from `judge-pack.yml` — provider default + mode default + cost ceiling + cache dir + prompt-version registry + run-after-pack-a1-only + per-bias-category strict opt-outs. Custom validator enforces unique (version, audience, category) composite key. |

Plus 7 type aliases (`JudgeProvider`, `JudgeMode`, `JudgeSeverity`, `JudgeBiasCategory`, `JudgeAudienceClass`, `JudgePromptCategory`) and 5 default constants. All `frozen=True, extra="forbid"` per the chassis standard.

### §3.2 CLI runbook — [`scripts/judge_brand_voice.py`](../../../../scripts/judge_brand_voice.py)

Paired runbook per [`akos-executable-process-catalog.mdc`](../../../../.cursor/rules/akos-executable-process-catalog.mdc) RULE 1. Subcommands:

- `--prose "..."` / `--file path` → single judge invocation; emits `JudgeVerdict` as JSON. Exit 0 unless mode=strict and verdict=fail (then exit 1).
- `--self-test` → mock provider round-trip (2/2 verdicts: pass exemplar + fail paraphrase exemplar) + cache-key determinism check. Used by the release-gate INFO row.
- `--bias-audit-summary` → structured stub indicating Strand C is forward-chartered; payload references `JudgeBiasMitigation` as the schema-locked surface so consumers know what shape to expect when audit data lands.
- `--print-config` → defaults + loaded pack as JSON; useful for operator runbook review.

Audience-class enforcement: the CLI's `_canonical_set_for(audience, override)` honors the dual-register contract from [`akos-brand-baseline-reality.mdc`](../../../../.cursor/rules/akos-brand-baseline-reality.mdc) — `external_register` reads public BRAND files only; `internal_register` reads `BRAND_BASELINE_REALITY_MATRIX.md` and CORPINT canonicals. The mock provider's 4 paraphrase exemplars cover the canonical brand-voice violations (`delve into`, `drill down into`, `navigate the landscape`, `unlock value`) the regex chassis is known to miss.
