# I48 P4 — Brand-aligned PDF mode (D-IH-48-H + D-IH-48-H1)

## What shipped

### `akos/dossier/pdf_render.py` (NEW; ~70 lines)

Thin wrapper around `akos.hlk_pdf_render.render_pdf_branded` (I27 P1; the proven WeasyPrint → fpdf2 → pandoc → markdown-sidecar fallback chain). NO new pipeline introduced; NO brand-token re-declaration; NO new akos/hlk_pdf_render.py changes.

| Public symbol | Purpose |
|:---|:---|
| `DOSSIER_TITLE` | "AKOS Operator UAT Dossier" (cover band title) |
| `DOSSIER_SUBTITLE_BY_MODE` | 3-mode subtitle map (snapshot / live / tier-b) per D-IH-48-C |
| `render_dossier_pdf(run, md_text, out_path) -> int` | Reuses `profile="dossier"` per D-IH-48-H1; passes `run_id` as `program_id`, `"UAT Dossier"` as `discipline`, `started_at` date as `issue_date`, `f"overall_status: {run.overall_status}"` as `status_label` |
| `_iso_date_from_started_at(iso)` | Extracts YYYY-MM-DD from ISO 8601 timestamp; falls back to today UTC on invalid input |

### D-IH-48-H1 ratification: profile="dossier" reused

Per D-IH-48-H1 (deferred from P0 to P4): the existing `profile="dossier"` from `render_pdf_branded` is reused unchanged. The cover band layout (program_id + discipline + issue_date + status_label) adapts cleanly to UAT context:

| ENISA dossier (I27 P4) | UAT dossier (I48 P4) |
|:---|:---|
| program_id = `PRJ-HOL-FOUNDING-2026` | program_id = `dossier-<uuid>` |
| discipline = `Asesoría Jurídica + Certificación` | discipline = `UAT Dossier` |
| issue_date = today | issue_date = run.started_at date |
| status_label = `P1-Audit ready` | status_label = `overall_status: PASS` |

**No new profile added** to `akos/hlk_pdf_render.py` (out of I48 scope per asset-classification.md); brand tokens stay in single SSOT.

### `scripts/render_uat_dossier.py` extension

`--format pdf` and `--format all` now invoke `render_dossier_pdf()`. Soft-success on fallback chain: when WeasyPrint+fpdf2+pandoc all unavailable, the markdown sidecar is written next to `dossier.pdf` (per the existing I22 P6 + I27 P4 contract). Manifest records `pdf_status: "sidecar_fallback"` when fallback fires; otherwise records `dossier.pdf` sha256 + byte_count.

## Performance

PDF mode adds ~3s to dossier render (WeasyPrint pipeline). Live operator smoke:
- snapshot mode + `--format pdf` → 80ms script + ~3s WeasyPrint = ~3.1s total
- snapshot mode + `--format all` → ~3s (md+pdf+html+manifest)

PDF byte size: ~80KB for the current 12-section dossier (1 page cover + ~3 pages body).
