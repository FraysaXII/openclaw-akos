# I48 P5 — Standalone HTML mode (D-IH-48-I; full markdown + brand CSS)

## What shipped

### `akos/dossier/html_render.py` extended (~250 lines total)

Replaces P1 minimal `<pre>` body with full markdown library invocation + brand CSS variables + collapsible `<details>` + dark-mode media query + print styles.

| New public symbol | Purpose |
|:---|:---|
| `render_dossier_html(run) -> str` | P5 end-to-end function: emits standalone HTML doc |
| `DOSSIER_HTML_CSS` | Full brand stylesheet (~3KB) — Inter typography + collapsible details + tables + dark-mode media query + print styles |
| `_render_section_body(markdown_body, use_markdown_lib)` | Converts section markdown to HTML via `markdown` library + MD_EXTENSIONS (same extensions as PDF chain) |
| `_strip_section_header(markdown_body)` | Removes leading `## Section N — Name` (already in `<summary>`; prevents duplicate) |

### Standalone-file invariant (R-48-3 + D-IH-48-I)

Per `tests/test_dossier_html_full.py::test_render_dossier_html_no_external_references`, the HTML output contains:
- NO `<script>` tag
- NO `https://cdn.*` reference
- NO `https://fonts.googleapis` reference
- NO `<link rel="stylesheet">` (all CSS inline)
- NO `src="http..."` reference

Plus a CSP `<meta http-equiv="Content-Security-Policy" content="default-src 'self' 'unsafe-inline'">` that limits XSS damage even if internal markdown accidentally embeds malicious HTML.

The operator can email `dossier.html` to anyone without security review.

### Brand-aligned styling (E9 single SSOT)

CSS variables derived from `BRAND_TOKENS_LIGHT` via `_brand_css_var_block()` (same as P1):
- `--c-accent-primary` (teal `hsl(168 55% 38%)`)
- `--c-accent-secondary` (amber `hsl(38 80% 50%)`)
- `--c-background` / `--c-foreground` / `--c-card` / `--c-secondary` / `--c-border` / `--c-muted-foreground`

Inter typography matches PDF chain. Dark-mode via `@media (prefers-color-scheme: dark)` overrides 6 token variables (CSS-only; no JS toggle). Print stylesheet ensures `break-inside: avoid` for `<details>` sections.

### Collapsible `<details>` per dossier-section-spec.md

Each section's `default_open_html` class attribute drives the `<details open>` attribute. Custom collapse marker via CSS `::before` pseudo-element (replaces default `<summary>` triangle with brand-colored chevron). When opened, section border switches to `--c-accent-primary`.

### Markdown library reuse

Uses the existing `markdown` library + `MD_EXTENSIONS = ["tables", "fenced_code", "toc", "sane_lists", "attr_list"]` (same as PDF chain via `akos.hlk_pdf_render.MD_EXTENSIONS`). Tables, code blocks, lists, and inline SVG (sparklines from P7) all render correctly.

NO new dependency added.

### Inline SVG sparkline support

`_render_section_body()` lets inline SVG fragments pass through unchanged (the markdown library treats them as raw HTML). When P7 wires sparklines into Section 11, they will render inline without an `<img src=>` reference.

### Section header de-duplication

`_strip_section_header()` removes the leading `## Section N — Name` from the body before passing to the markdown library, preventing duplicate H2 (the section header is already shown in `<summary>`).

### `scripts/render_uat_dossier.py` cleanup

- Removed inline `_render_minimal_html()` placeholder + `_html_escape()` helper (now in `html_render.py`)
- `--format html` and `--format all` now invoke `render_dossier_html(run)`

## R-48-3 mitigation (HTML XSS surface)

Strategy:
1. **No JS framework** (D-IH-48-I) — no Alpine.js / htmx / React / Vue
2. **Inline SVG sparklines** (no `<img src>` references)
3. **CSP meta tag** restricts script-src to `'self' 'unsafe-inline'` (no remote scripts loadable)
4. **Internal-content trust** — section bodies are GENERATED by sources.py + runner.py; no operator input passes through (we control the markdown)
5. **Sticky test** `test_render_dossier_html_no_external_references` enforces the standalone invariant in CI

## Performance
