# I45 P6 — Tier B weekly schedule + cost ceiling kill switch

**Phase:** P6 (Tier B GitHub Action workflow + --tier flag + --max-spend kill switch + --regression-pp threshold)
**Closes:** I45 P6 + evidence-matrix E7 (Tier B effectively dormant) + R-45-3 mitigation operationalized.
**Date:** 2026-05-01

## Actions

1. **`scripts/eval.py` extensions**:
   - `--tier {A,B}` flag (default A; B requires `AKOS_RECORD_LIVE=1`)
   - `--max-spend <USD>` flag (defaults to `MAX_TIER_B_USD` env or 5.0)
   - `--regression-pp <pp>` flag (default 5.0; D-IH-45-D)
   - Tier B preflight: hard-stops with explanation + exit 2 if `AKOS_RECORD_LIVE` unset
   - Tier B post-run: rolls up total cost across all rows; kills if > ceiling; marks rows FAIL if delta_pp regresses past `--regression-pp`

2. **`.github/workflows/eval-tier-b.yml`** — the actual scheduled CI:
   - **Cron**: `0 6 * * 1` (Monday 06:00 UTC, post-weekend, pre-billing-cycle)
   - **`workflow_dispatch`** with operator-set inputs: `max_spend_usd`, `regression_pp`, `skill_filter`
   - **Matrix**: `[cheap, flagship]` model tiers (operator wires actual model_ids via repo secrets)
   - **Repo-var gated**: `if: ${{ vars.AKOS_TIER_B_ENABLED == 'true' }}` — zero-cost until operator opts in
   - **Per-tier preflight**: SKIPs gracefully when API key not in secrets (no confusing failures)
   - **3 eval runs per tier**: Tier A baseline (always), Tier B replay (live), Tier B adversarial (live)
   - **Artifact upload**: scorecard JSONs retained 90 days
   - **Job summary**: per-failure surfacing in GitHub UI via `$GITHUB_STEP_SUMMARY`
   - **Timeout**: 20 min (well below the 60 min default, leaving headroom for slow models)

3. **`config/verification-profiles.json` new profile `eval_harness_smoke`** (Tier A only):
   - 4 steps: `--mode all` + `--mode replay` + `--mode adversarial` + `lint_cassette_pii.py`
   - Suitable for `pre_commit` profile inclusion (no LLM cost)
   - Operator can wire it into pre_commit independently of P6 if desired

4. **11 new tests** in `tests/test_eval_tier_b.py`:
   - Cost-control guard (5 tests): preflight blocks without env var; banner appears with env var; --max-spend flag respected; MAX_TIER_B_USD env respected
   - Regression threshold (2 tests): synthetic 12pp drop fails at --regression-pp 5; 3pp drop trips canary-2 only (not tier_b_regression at threshold 5)
   - GitHub workflow YAML (3 tests): file exists with cron + dispatch + matrix; model_tier matrix has cheap+flagship; AKOS_TIER_B_ENABLED gating present
   - Verification profile (1 test): eval_harness_smoke registered with 4 expected steps

## Sample workflow run preview (operator-side)

When operator sets `AKOS_TIER_B_ENABLED=true` repo variable + `OPENAI_API_KEY` + `ANTHROPIC_API_KEY` secrets, the scheduled run will:
1. Check out the repo, install Python deps
2. Pre-flight cheap and flagship tiers (skip if respective key missing)
3. Run Tier A baseline first (deterministic; must pass before Tier B even attempts)
4. Run Tier B replay against the matrix tier
5. Run Tier B adversarial sweep against the matrix tier
6. Upload all scorecard JSONs as artifacts (90-day retention)
7. Surface failures in the job summary

## What this does NOT do (deferred)

- **Live LLM cassettes still don't exist**: P2 ships only deterministic + P6 stub `live_llm` cassettes. The first real Tier B run will be a no-op until operator records live cassettes via `AKOS_RECORD_LIVE=1 py scripts/eval.py record --kind live_llm ...`. The infrastructure is ready; the data is operator-side.
- **No rolling-baseline cost regression** — P4 enforces vs static POLICY_REGISTER ceiling. The `compliance.eval_run` mirror (P4 DDL) becomes the rolling-baseline source once it accumulates 4 weeks of data. P7 or I47 wires the rolling logic.
- **No automated alerting**: workflow surfaces failures in GitHub UI; no Slack/email post yet (operator-side; can wire via existing GitHub Action notifiers).

## Risks resolved or deferred

- **R-45-3 (Tier B costs balloon)**: full mitigation in place. Per-run cost ceiling + per-skill cost_ceiling policies + operator opt-in gate + matrix tier separation = 4 layers of cost control.
- **R-45-1 (cassettes go stale)**: workflow runs weekly → if a cassette fails replay due to staleness (90+ days unchecked), the alarm fires inside the weekly run.
