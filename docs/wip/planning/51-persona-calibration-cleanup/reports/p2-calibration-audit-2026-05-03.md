# I51 / P2 — Calibration audit (baseline + 13-outlier remediation report)

**Phase scope:** run the baseline calibration audit, capture the persona-level
delta_pp signal, and write a deterministic remediation plan against the
13 outliers under the **D-IH-51-A** default path (per-persona
`target_difficulty_band` column). The phase is **read-only** with respect to
the CSV — no scenario mutation occurs here. The CSV mutation lands in I51/P3
under the **G-51-1** operator gate.

**Phase non-scope:** the actual `PERSONA_SCENARIO_REGISTRY.csv` edit, the
`scripts/validate_persona_scenario_registry.py` check addition, the
`akos.eval_harness.persona.calibration_distribution` wiring, and the
re-calibration. All four are I51/P3 deliverables under one operator-gated
commit.

| Archetype | Personas | Proposed band | Rationale shorthand |

| **Press / moderate-saturated** | PRESS | 10/60/25/5 | 23.6pp moderate skew is structural |
| **Hard-saturated partners/customers** | EXISTING-PARTNER, EXISTING-CUSTOMER, PARTNER-JOINT-EQUITY, RANDOM-INBOUND | 10/35-40/45/5-10 | Operationally hard prompts dominate; rebalance would be performative |

| **Trivial-skewed funnel-top / operator** | OPERATOR, INVESTOR-COLD | 15/40/35/10 | Look-up + first-contact prompts skew trivial by design |
| **Thin-N (n=6) hard-skewed** | CUSTOMER-SERVICE-PROSPECT, IDEA-PROPOSER | 0/35/55/10 | Sample size too small to balance without bloat; codify the skew |

The two thin-N personas are flagged as the explicit candidates for the
"≤ 2 personas may remain outside tolerance" allowance at the I51/P3 exit.
