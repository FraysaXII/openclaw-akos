During the post-closure self-review the operator surfaced a real SSOT collision in the Wave M deliverables. Three surfaces named the same 12 regression dimensions three different ways:

| Surface | What it said DIM-01 was | What it said DIM-02 was |
|:---|:---|:---|
| Canonical [`INTER_WAVE_REGRESSION_DISCIPLINE.md`](../../../references/hlk/v3.0/Admin/O5-1/People/canonicals/INTER_WAVE_REGRESSION_DISCIPLINE.md) §2 table | `decision_lineage` — FK-resolve DECISION_REGISTER ↔ canonical frontmatter | `forward_charter_carryover` — glob prior canonicals + resolve forward_charters |

| Pydantic [`akos/hlk_inter_wave_regression.py`](../../../../akos/hlk_inter_wave_regression.py) `VALID_DIMENSION_CODES` | `DIM-01-CLOSING-WAVE-SURFACES` — checks wave-touched files still on disk | `DIM-02-SIBLING-INITIATIVE-STATUS` — sweeps INITIATIVE_REGISTRY status enum |
| Runbook [`scripts/inter_wave_regression_sweep.py`](../../../../scripts/inter_wave_regression_sweep.py) `PROBE_REGISTRY` | Same as Pydantic (matched code-side) | Same as Pydantic (matched code-side) |

Reading-order failure mode: any cursor session reads the always-applied cursor rule first (RULE 1 table), then opens the canonical (§2 table), then asks the runbook to execute — and discovers the runbook checks something completely different. The cursor rule + canonical actively misled the agent about what the sweep verifies.

### 12.2 The two options + operator ratification

The agent surfaced both options via `AskQuestion` inline-ratify gate:

- **Option A — code-wins**: rewrite the canonical + cursor rule (12 dimension names + 12 probe heuristic rows) to match the implementation. Smaller diff; preserves the implementation that ran 3 live sweeps cleanly.
- **Option B — doctrine-wins-engrave-properly**: rewrite the Pydantic enum + 12 probe functions + tests to match the canonical + cursor rule. Larger diff; preserves the doctrine bit-for-bit (the documentation is authoritative).

Operator ratified **Option B + Wave-M.5-hotfix-rework-now classification** (atomic hotfix on top of `e82a0ae` + `07c5a10`; not a new initiative). The choice aligns with the same precedent applied at I72 P4 Cluster B (engrave-properly OVERRIDE → mint 4 full canonicals not stubs): when code and doctrine collide, doctrine wins; the canonical is the authoritative read-path for any future agent.

### 12.5 Self-discipline lesson for future agents

> When an always-applied cursor rule + canonical table + code Literal enum collide on the same governed enum, **always pick doctrine + rewrite the code**. The cursor rule is the codified agent default reading-path; misleading the agent at that surface costs orders of magnitude more in downstream confusion than rewriting an enum + 12 probe functions + 49 tests.
