### SQL view shape

Per plan body: `CREATE VIEW governance.initiative_program_rollup_view` joining:
- `compliance.initiative_registry_mirror` (LEFT side; PK `initiative_id`)
- `compliance.program_registry_mirror` (RIGHT side; PK `program_id`)
- Join arity: `string_to_array(program_anchors, ';')` against `program_id` — one initiative produces N rows in the view (one per anchor).

RLS posture per plan body + I66 P6 precedent: `service_role` + `authenticated SELECT-only`; no anon access (initiative metadata is `confidential` per CANONICAL_REGISTRY row 31).
