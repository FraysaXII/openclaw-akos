# I-NN-RESEARCH-AREA-CROSS-AREA-TOPIC-INTENT-IMPROVEMENT — extend Research area to operate cross-area topic + intent intelligence as a sustained function

> **Spawned by D-IH-81-Z** (operator novel framing at Bundle B Strand-2 ratify gate 2026-05-24). Operator surfaced the Spain-strategy intelligence request mid-batch (Strand-2 disambiguation context: AT-Pymes + BBVA + PAE network framing required ad-hoc deep research; operator wanted "to be knowledgeable about Spain strategies… a news feed, a radar, deep research with intelligent intelligence processing"). Operator then reframed: *"this is a research request and our current architecture could not secure it i think. That's why i ask this challenge and i expect you to link it to a research area improvement (with the cross area topics and intents in it)."* This candidate names the gap, frames the improvement, and uses Spain-strategy as the worked-example activator.

## 1. The gap (one paragraph)

The current Research area substrate has three load-bearing canonicals — **INTELLIGENCEOPS_REGISTER.csv** (per-target intelligence operations), **SOP-IO_INTELLIGENCE_REPORT_001** (HUMINT-style report craft), **SUBSTRATE_LANDSCAPE_DOCTRINE** (research substrate inventory) — but none of them carry the shape the operator named: **sustained cross-area research on a topic + intent matrix that informs how every area engages that topic**. IntelligenceOps is target-class-driven (regulator / competitor / media / recruiter); SOP-IO_INTELLIGENCE_REPORT is per-engagement HUMINT report; SUBSTRATE_LANDSCAPE is substrate-inventory-driven. The Spain-strategy ask is **none of those**: it is *"hold an ongoing research function on a topic (e.g. Spain market + regulatory + funding + talent landscape) cut by intents (e.g. customer-acquisition / talent-hiring / partner-discovery / regulatory-readiness / banking-relationships) so that every area (Marketing / Operations / People / Legal / Finance / Tech Lab) can pull from a shared current substrate when they need to engage that topic."* That shape does not exist in the canonical layer today.

## 2. Why the operator's framing is load-bearing

The operator's framing names two structural pieces missing from the current architecture:

1. **Topic dimension** — a long-running research subject (country/region; market vertical; technology family; regulatory regime; talent pool; capital landscape) that warrants sustained attention rather than per-engagement HUMINT bursts.
2. **Intent dimension** — the cross-area uses each consuming area has for the topic. The same topic surfaces in Marketing (positioning + go-to-market) + Operations (vendor + banking + administrative substrate) + People (talent pool + cultural fit) + Legal (jurisdiction + compliance regime) + Finance (FX + banking + tax) + Tech Lab (regional infra + data residency). Each area's intent is different; the underlying topic substrate is shared.

Cross-area = the canonical research output is consumed by multiple areas, not owned by one. Topic + intent = the matrix that lets each consuming area filter the substrate to what they need. **This is a new Research area discipline shape**, not a register row.

## 3. Spain-strategy as worked example (the activation surface)

The operator named the gap via Spain-strategy specifically (mid-Strand-2 because AT-Pymes + BBVA + PAE-network disambiguation forced ad-hoc research to ground the FINOPS counterparty rows). Spain-strategy is the worked example because:

- **Topic**: Spain — incorporation routes (CIRCE/PAE); banking landscape (BBVA partnership tier); legal/fiscal regime (autonomo + societario); talent + remote-work culture; EU funding programs; market segments (SMB-friendly per AT-Pymes positioning); language + regional differentiation.
- **Intents (cross-area)**:
 - **Operations** — gestoria + banking + administrative substrate (already partial: AT-Pymes + BBVA rows in FINOPS register).
 - **Legal** — autonomo societario + Modelo 036/037 + IRPF + IVA + ENISA pathway.
 - **People** — Spanish talent pool + remote-work norms + visa + relocation if any.
 - **Marketing** — Spanish-market personas + Madeira positioning vs mainland Spain positioning.
 - **Finance** — EUR-banking + EU funding lines + Spanish tax credits + R&D deductions.
 - **Research** — Spanish competitive landscape + Spanish research-funding bodies (CDTI etc.) + Spanish AI/ML community.
 - **Tech Lab** — Spanish data residency + Spanish cloud regions + EU GDPR posture.

If the cross-area topic + intent shape existed today, all of this would live in one Research-owned canonical surface that every other area's SOPs cross-reference when their work touches Spain. Today, it lives in scattered notes + per-engagement intelligence bursts.

## 4. The proposed improvement (forward-charter shape)

When this candidate activates as an initiative, it will (subject to ratification at initiative-charter time):

1. **Mint a new canonical shape** — `RESEARCH_TOPIC_INTENT_MATRIX_REGISTRY.csv` (or named equivalent) under `Research/Intelligence/canonicals/dimensions/` carrying rows like `(topic_id, intent_id, consuming_area, lead_researcher, sustain_cadence, source_substrate_pointer, last_updated, confidence_level)`.
2. **Mint a research-area discipline doctrine** — e.g. `CROSS_AREA_TOPIC_INTENT_DISCIPLINE.md` under `People/canonicals/` (per `akos-people-discipline-of-disciplines.mdc` RULE 1 — People owns cross-area methodology) that names the shape + cadence + cross-area-handoff pattern.
3. **Author the Spain-strategy worked-example substrate** — `docs/references/hlk/v3.0/Admin/O5-1/Research/Intelligence/canonicals/topics/SPAIN_STRATEGY_SUBSTRATE.md` (or equivalent) as the first concrete topic + intent matrix populated row.
4. **Wire cross-area consumption** — every consuming area's relevant SOP cross-references the topic substrate (Marketing SOP for Spain personas references the topic substrate; Legal SOP for Spanish incorporation references it; FINOPS counterparty rows for Spanish vendors reference it). Cross-references are validated by `validate_hlk.py` umbrella extension when the substrate exists.
5. **Forward-charter sister topics** — once the Spain-strategy worked example proves the shape, sister topic substrates follow (Madeira-strategy, EU-AI-Act-strategy, US-market-strategy, etc.) at operator cadence.
