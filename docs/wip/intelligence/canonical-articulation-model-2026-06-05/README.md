# Canonical Articulation Model — research action (the Singularity)

**Decision:** `D-IH-95-A` · **Initiative:** I95 (derived from I94) · **Authored:** 2026-06-05
**Confidence:** Keter · **Sources:** 115 (51 internal / 64 external; validator PASS)

The research action that solves the operator's never-cracked **linking problem**: how every
articulable canonical (role, process, topic, project, workstream, engagement, component, capability,
decision, …) links to every other **and across its own internal layers** — and what their
end-to-end pairing (**"the Singularity"**) actually is.

## The one-line answer

The Singularity is the **Digital Twin of the Organization (DTO)** = an **enterprise ontology**
(the **Holistika Canonical Articulation Model, HCAM**) over a **governed knowledge graph** (Neo4j,
typed via an **ISO GQL graph type**) over the **CSV SSOT**. HCAM uses ArchiMate's **closed ~10
relationship verbs** (composition / assignment / realization / serving / influence / …) so the
graph stops forking the same relationship per type. Owned **Data-federated** via the Semantic
Council (`D-IH-95-D`) — *not* a new role; the sellable corporate-intelligence quality bar is a
**Research** workstream (`D-IH-95-G`, the KM Officer seat). [corrected 2026-06-06 — superseded the
earlier "Knowledge Manager under People" framing per the 53-source data-governance findings.]
