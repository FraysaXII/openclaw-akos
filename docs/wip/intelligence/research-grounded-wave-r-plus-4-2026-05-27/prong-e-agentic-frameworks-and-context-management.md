We are really in a question world, not a prompt world."*

**Why this matters for Holistika:** The cursor rule `akos-inline-ratification.mdc` + its paired skill `inline-ratify-craft` already operationalise this for **agent → operator** ratify gates. ### Insight E-4 — "Read what the model says" (Theo; CL4-HIGH; RANK 3)

**Verbatim claim** (Theo `How I code with AI changed a lot`, ~22:00): *"The important thing here, and I know this is hard for people, you have to read what it says. I know people tend to gloss over the text the model puts out and instead read the code it puts out... that's entirely backwards. You have to read what it says."*

**Why this matters for Holistika:** This is the load-bearing failure-mode the inline-ratify discipline (`akos-inline-ratification.mdc`) was designed to prevent. The "scroll-past-the-text-go-to-the-code" anti-pattern is the SAME failure mode as "skip-the-rationale-pick-the-option" in inline-ratify gates. Holistika's inline-ratify-craft skill Principle 2 *"every option carries rationale embedded inline"* exists BECAUSE the operator + agents alike are tempted to skip reading. The discipline is congruent. The investor-brief framing can cite this as evidence the methodology grapples with a real industry failure mode that the dev community is now naming explicitly.

### Insight E-5 — "Composer 2.5 + Kimmy K2.5 distillation evidence" (Theo `Cursor just crushed Claude Code`; CL4-HIGH; RANK 2)

**Verbatim claim** (Theo `Cursor just crushed Claude Code`, ~02:00 + ~25:00): *"Composer 2.5 is built on the same open-source checkpoint as Composer 2, which was Moonshot's Kimmy K2.5... Showing that you can distill an open weight model to be close to that level of intelligence in coding task specifically is both super cool to see but also kind of shows that all of the work for making models good at code is post training in RL largely."*

**Why this matters for Holistika:** The trend toward **specialized model distillation** + **RL post-training** validates the operator's MADEIRA 2023/24 thesis that **framework + context + discipline** (not raw model capability) is the lasting competitive advantage. A distilled-Kimmy-K2.5 at ~5% the cost of GPT-5.5 with comparable code-bench scores means:

1. The "best model" is not the moat; the **discipline around model usage** is.
2. Holistika's investment in framework-agnostic + context-rich + discipline-grounded methodology PRECEDED this industry shift by 2 years.
3. The investor narrative should de-emphasize "we use the latest models" (zero moat — anyone can subscribe) and emphasize "we have built the discipline that makes ANY model perform well on R&D-class tasks" (durable moat).
