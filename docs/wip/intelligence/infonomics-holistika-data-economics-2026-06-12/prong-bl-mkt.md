# Prong BL-MKT — Marketing / Reach consumer

## Narrative findings

### 1. Brand canon is Holistika's highest-density "information moat" inventory

The Marketing prong carries **41 CORPINT rows** — the deepest internal harvest. **Brand Architecture** (`SRC-INF-INT-070`), **Baseline Reality Matrix** (`SRC-INF-INT-071`), **Copywriting Discipline** (`SRC-INF-INT-067`), and **Counterparty README Contract** (`SRC-INF-INT-005`) encode **non-rival information assets**: dual-register vocabulary, template registries (`SRC-INF-INT-031`), and multilingual contracts (`SRC-INF-INT-030`) that reduce marginal cost of each new external surface. **Brand Template Registry maintenance SOP** (`SRC-INF-INT-032`) prices upkeep explicitly in process time if not in EUR.

This is Infonomics *manage* at scale — the asset appreciates when reused, depreciates when drift validators fire.

### 2. Cobranding and reach adapters imply attribution economics

**Cobranding Pattern** (`SRC-INF-INT-029`) and reach-side adapter registries (CRM `SRC-INF-INT-017`, email `SRC-INF-INT-018`, communication `SRC-INF-INT-016`, attribution `SRC-INF-INT-015`) map **channel-specific data flows**. External Amplitude/Heap APIs (`SRC-INF-EXT-307`, `SRC-INF-EXT-308`) show how industry prices **event-level audience data** — Holistika lacks a unified **cost-per-attributed-insight** row tying MarTech spend to FINOPS.

### 3. MarTech stack skepticism challenges ROI narratives

**MarTech stack complexity skeptic** (`SRC-INF-EXT-494`) and related tradeoff rows (CRM customization debt `SRC-INF-EXT-262`, CDP confusion `SRC-INF-EXT-265`, marketing automation overpromise `SRC-INF-EXT-264`) warn that **tool proliferation destroys economic value** faster than analytics dashboards show. **Brand DO/DON'T** (`SRC-INF-INT-073`) and voice pattern canon reduce rework cost — an internal substitute for buying more MarTech.

### 4. Knowledge moat vs generative commodity content

Brand disciplines (AV charter `SRC-INF-INT-066`, design charter `SRC-INF-INT-068`, English/French/Spanish patterns `SRC-INF-INT-074`–`SRC-INF-INT-076`) protect **differentiated narrative stock**. External Notion AI patterns (`SRC-INF-EXT-373`) and agent platform overviews (`SRC-INF-EXT-390`, `SRC-INF-EXT-391`) threaten substitute **good-enough** copy — economic value shifts to **governed register compliance**, not word count.

### 5. Excalidraw and design-to-code links carry build economics

Excalidraw docs (`SRC-INF-EXT-357`) and Figma-adjacent harvest under Data prong (`SRC-INF-EXT-471`) connect Marketing surfaces to Tech build cost — MarTech ROI incomplete without **render pipeline** COGS (Tech SOP `SRC-INF-INT-178`).

## Infonomics hook

**Economic levers for Marketing/Reach:** brand canon as depreciating/appreciating IP stock; template registry as reuse multiplier; adapter registries as channel COGS map; attribution data as monetizable event inventory (with skeptic discount `SRC-INF-EXT-494`).

**Holistika delta vs external Infonomics posture:** industry prices **events and seats**; Holistika prices **governance and register integrity** — stronger long-term moat, weaker near-term analytics ROI story unless FINOPS ties reach spend to closed engagements (`SRC-INF-INT-047` engagement templates under Ops).

**Govern options (ranked):**

1. **MarTech COGS row in UNIT ECONOMICS doc** (`SRC-INF-INT-111`, Ops) fed by adapter registry status columns — narrative-first, no CSV gate. *(recommended for P5)*
2. **`BRAND_TEMPLATE_REGISTRY` economic metadata** (reuse count, drift repair minutes) — extends existing maintenance SOP (`SRC-INF-INT-032`).
3. **Consolidate attribution adapters before new MarTech** — operational de-stack per skeptic (`SRC-INF-EXT-494`); savings as Infonomics win, not new doctrine.
4. **External audience-data product SKU** — highest legal/compliance overlap; schedule post-P6 vault tranche, not dropped.
