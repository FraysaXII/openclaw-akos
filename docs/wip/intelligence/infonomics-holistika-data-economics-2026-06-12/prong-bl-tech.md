# Prong BL-TECH — Tech / System Owner consumer

> **Baseline prong:** `BL-TECH` (O5-1 Tech / System Owner)  
> **Ledger coverage:** 50 rows (15 CORPINT + 35 OSINT); **7 skeptic** voices  
> **Downstream decision:** **D-INF-ECON** (P5 govern)

## Narrative findings

### 1. TechOps doctrine and CI baseline encode verification as economic infrastructure

**TechOps Discipline** (`SRC-INF-INT-181`), **SOP-CICD_BASELINE_001** (`SRC-INF-INT-174`), **SOP-HLK_TOOLING_STANDARDS_001** (`SRC-INF-INT-176`), and **SOP-TECH_SYSTEM_RELIABILITY_001** (`SRC-INF-INT-180`) treat validators, Playwright baselines, and release gates as **non-optional factory equipment**. Each verify profile step is a **recurring information quality tax** — cheap versus production incident, expensive if unpriced in Infonomics. **SOP-TECH_DATAOPS_QUALITY_001** (`SRC-INF-INT-179`) explicitly bridges Tech to DataOps economics.

### 2. Agentic platform harvest shows token and tool-call as variable COGS

Infonomics-core OSINT includes **Cursor hooks** (`SRC-INF-EXT-342`), **Cursor agent tools** (`SRC-INF-EXT-376`), **GitHub Copilot agent mode** (`SRC-INF-EXT-350`), **LlamaIndex workflows** (`SRC-INF-EXT-349`), **Google ADK** (`SRC-INF-EXT-377`), **AWS Bedrock agents** (`SRC-INF-EXT-378`), and **Azure AI Agent Service** (`SRC-INF-EXT-379`) — industry converges on **metered agent runtime** as opex. Holistika **MADEIRA** registries live under Envoy (`SRC-INF-INT-013`) but settlement must appear on Tech+FIN dashboards jointly.

### 3. MCP and render pipeline SOPs price integration surface area

**SOP-MCP_SERVER_DEFINITION** (`SRC-INF-INT-177`) and **SOP-RENDERING_PIPELINE_GOVERNANCE_001** (`SRC-INF-INT-178`) expand callable **information interfaces** — each MCP tool and render path adds maintenance + security review cost. **SOP-EXTERNAL_RENDER_GATE_PROMOTION_001** (`SRC-INF-INT-175`) prevents cheap external publish that would destroy brand information asset value (Marketing cross-ref `SRC-INF-INT-071`).

### 4. Skeptic cluster warns against platform engineering and DevOps silver bullets

Seven skeptic rows: **DevOps handbook** (`SRC-INF-EXT-095`), **toil reduction SRE** (`SRC-INF-EXT-099`), **DevOps silver bullet critique** (`SRC-INF-EXT-270`), **CI/CD breach catalog** (`SRC-INF-EXT-310`), **Thoughtworks tech radar** (`SRC-INF-EXT-316`), **microservices tradeoffs** (`SRC-INF-EXT-324`), **platform engineering skeptic** (`SRC-INF-EXT-499`). Collective message: **platform teams and CI expansion consume budget** without guaranteed throughput gain — DORA (`SRC-INF-EXT-302`, Ops prong) must prove value.

### 5. Counterparty brief corpus is pre-priced information packaging for external audiences

Tech prong CORPINT includes adviser/investor/ENISA/recruiter/partner/sales **counterparty briefs** (`SRC-INF-INT-146`–`SRC-INF-INT-151`) — each is a **reusable information product** with audience-specific economic purpose. They embody Infonomics *monetize* at GTM layer even before formal SKU exists.

## Porter — four forces + competition synthesis

| Force | Finding (cite `SRC-*`) |
|:---|:---|
| **Supplier power** | Hyperscaler agent services (`SRC-INF-EXT-378`, `SRC-INF-EXT-379`) and IDE vendors (`SRC-INF-EXT-342`) control token pricing; CICD baseline (`SRC-INF-INT-174`) locks GitHub/Vercel/Render economics. |
| **Buyer power** | Internal operators demand fast verify (`SRC-INF-EXT-099`); external investors read investor brief (`SRC-INF-INT-148`) — buyers of Holistika story expect **evidence of disciplined spend**. |
| **Threat of substitutes** | Single-vendor low-code agents (`SRC-INF-EXT-387` Copilot Studio) substitute custom MCP mesh; skipping Playwright (`SRC-INF-INT-174`) substitutes manual browser UAT at higher reputational risk. |
| **Threat of new entrants** | AI coding CLI hype (`SRC-INF-EXT-496`, Envoy prong) — entrants ship features without governance stack; compete on demo velocity until reliability fails. |
| **Competition synthesis** | Holistika Tech economics = **governed automation tax** (verify + MCP + render gates) vs **agent variable spend**. Competitive advantage when total cost is visible and below cost of incident + rebrand + legal rework; skeptics (`SRC-INF-EXT-499`, `SRC-INF-EXT-270`) demand proof via DORA metrics, not platform headcount. |

## Infonomics hook

**Economic levers for Tech/System Owner:** verify profile minutes as factory opex; agent token/API as variable COGS; MCP/render surface count as integration debt; CICD baseline as throughput asset; counterparty briefs as packaged information SKUs.

**Holistika delta vs external Infonomics posture:** cloud FinOps covers **infra**; Tech stack adds **governance automation** and **agent runtime** — dual opex layers often invisible in one dashboard. Laney measure (`SRC-INF-EXT-477`) needs token + validator + mirror emit in one view.

**Govern options (ranked):**

1. **SUBSTRATE_REGISTRY + MADEIRA registries: add `cost_class` / monthly burn rollup** (`SRC-INF-INT-011` lineage, `SRC-INF-INT-013`) — extends existing enum patterns; FINOPS recon consumes. *(recommended)*
2. **Agent spend cap policy in TECHOPS_DISCIPLINE** (`SRC-INF-INT-181`) referencing Bedrock/Cursor OSINT — soft gate before hard budget.
3. **Platform engineering pause** until DORA (`SRC-INF-EXT-302`) baseline recorded — responds to skeptic (`SRC-INF-EXT-499`); scheduled review, not dropped capability.
4. **Outsource verify to vendor CI only** — lowers fixed cost, loses intersubjective verifiability (area-completeness precedent); high risk.
