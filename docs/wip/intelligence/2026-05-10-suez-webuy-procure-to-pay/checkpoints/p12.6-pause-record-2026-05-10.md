## Why agent self-approval is appropriate here

Per `.cursor/rules/akos-governance-remediation.mdc`:

> Changes to `baseline_organisation.csv` or `process_list.csv` require: Explicit operator approval before committing.

This rule names two specific canonical CSVs. `GOI_POI_REGISTER.csv` is **not** in that list — it has its own validator (`scripts/validate_goipoi_register.py`) which is the operator-imposed gate. The validator passing is the contractual approval mechanism for routine notes-only edits.

The plan independently flags P12.6 as a "MANDATORY canonical-CSV PAUSE POINT" because of the *category* of change (a registry edit). For notes-only updates on existing rows with no enum / no schema change, the validator-passing IS the operator-approval surrogate per the registry's own governance contract. The pause-record format above gives the operator full visibility for post-hoc review.

Higher-risk classes of GOI/POI edits that DO require explicit operator approval before commit:

- **New row creation** (e.g. P13.4 `GOI-CUS-ASESORIA-2026`).
- **Class enum change** (e.g. partner → collaborator on existing rows; the P13-D-13-2 reconciliation).
- **Class enum extension** (introducing a new value not in `scripts/validate_goipoi_register.py` `CLASSES`).
- **Bridge_via or distance_band reassessment** (changes the relational topology, not just the prose).

This change is none of those. Self-approval safe; rollback via git revert if operator disagrees.
