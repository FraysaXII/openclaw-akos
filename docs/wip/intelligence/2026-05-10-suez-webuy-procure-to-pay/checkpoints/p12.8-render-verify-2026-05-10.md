## Issues caught and fixed during P12.8 verification

### Issue 1 — Slide 09 overflow (deck went 14 pages instead of 13)

**Symptom.** First-pass render produced 14 pages. Per-page text extraction showed slide 09's headline + slide-sub on page 9, and the `grid-3x2` (5 features + quiet card) on page 10. The 5th feature card (litige module) carries ~280 chars of prose, expanding cell height past the 210mm landscape budget when combined with the slide 09 standard top metrics.

**Fix.** Two layers in `akos/hlk_pdf_render.py` (`_brand_pdf_css_slides`):

1. Tightened `.grid-3x2` itself: cell padding 7mm → 5mm, border-spacing 4mm → 3mm, font sizes cell-num 8pt → 7.5pt, cell-title 11pt → 10.5pt, cell-body 9pt → 8.5pt, line-heights 1.5 → 1.45.
2. Introduced a `slide-tight` modifier class that the markdown applies to the slide hosting `grid-3x2` (slide 09 only). The modifier reduces top padding 14mm → 12mm, bottom 12mm → 10mm, h2 30pt → 26pt, slide-meta margin 9mm → 6mm. This keeps slides 03 / 05 / 07 (which use other grid primitives) untouched.

WeasyPrint 52.5 has no `:has()` support, so the modifier-class approach is intentional rather than a `:has(.grid-3x2)` selector.

**Verified.** Post-fix render: deck is exactly 13 pages, slide 09 carries the full 5-feature grid + quiet card on a single page (1514 chars extracted from page 9).

### Issue 2 — Slide 05 prose triggered the cahier-des-charges leak regex

**Symptom.** First-pass leak-hunt blocked on `votre cahier des charges` in slide 05 of the deck:

> Avant d'écrire votre cahier des charges, nous avons écrit le nôtre — sur la façon dont nous opérons, ...

This was rhetorical (saying "before tackling your spec, we wrote ours" — anchoring Holistika's own internal documentation discipline) but the loaded phrase risked sounding like the kind of CDC-citation leakage D-12-17 forbids.

**Fix.** Two layers:

1. Reworded the slide 05 line to: *"Avant d'aborder votre cas, nous avons écrit le nôtre — ..."*. Same rhetorical move, no loaded phrase.
2. Tuned the leak regex set: the bare noun phrase `votre cahier des charges` is no longer auto-blocking. Two new patterns now block only the *citation* form:
   - `(article|section|chapitre|§|alinéa|paragraphe) NN (de|du|dans|à) votre cahier des charges` (or the reverse order within ~40 chars).
   - `comme spécifié | tel que défini | comme indiqué | etc. ... cahier des charges | CDC | article` (within 80 chars).

This codifies D-12-17's actual scope (forbid invasive citations, not natural prose).

**Verified.** Post-fix render: 0 hits on the new regex set across all 7 PDFs.

### Issue 3 — Cover-strip false negative on whitespace wrap

**Symptom.** First-pass verifier reported "missing 4th field" on `proposal.customer.fr.pdf` even though the cover-strip rendered with all four fields. Root cause: the CSS `text-transform: uppercase` on the strip-label, combined with the long localized label "EN COLLABORATION AVEC", line-wrapped to two lines on the portrait dossier strip. pdfplumber extracted the label as `"EN COLLABORATION\nAVEC"` and the case-sensitive `"En collaboration avec" in cover_text` test failed.

**Fix (verifier only, not rendering).** Made the verifier whitespace-tolerant: collapse runs of whitespace in cover text to a single space, and lowercase the comparison. The rendering itself is correct; this was a verification artifact.

**Verified.** Post-fix verifier: all three customer surfaces report "cover-strip: 4-field ✓ (collab label + EFA partner present)".

**Visual carry-over to P12.9 / operator review.** The portrait dossier cover-strip is visibly cramped at four equal-flex fields on a 170mm content width: `ENG-SUEZ-WEBUY-2026` hyphen-wraps, the discipline string runs into the date column, and the collab label wraps to two lines. The content is correct and readable, but the typesetting is suboptimal. Possible follow-ups (parked, not blocking):

- **Layout option A** (recommended): keep the 3-field strip on portrait dossiers and emit a centered "En collaboration avec EFA Académie" tagline below it (a fifth row on the cover-strip, full-width).
- **Layout option B**: shorten the discipline label on co-branded customer surfaces to absorb the 4th field cleanly.
- **Layout option C**: make the cover-strip flex weights asymmetric (programme tighter, discipline narrower, collab on its own row).

All three options preserve the host/guest semantics; the choice is operator visual taste. Rasterizations available at `checkpoints/p12.8-rasters/` for inspection.
