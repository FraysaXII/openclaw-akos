# docs/wip/intelligence/ — Tier 1 WIP (Research-owned, cross-area)

> Per WORKSPACE_BLUEPRINT_HOLISTIKA §17 + D-IH-70-O (3-tier WIP topology) + RESEARCH_AREA_CHARTER §5. **Research area owns this folder** as Tier 1: cross-area research, knowledge-base development, ontology drafts, classification-lattice candidates, taxonomy proposals, methodology-pillar exploration. Curated by KM Officer.

## What lives here

- **Cross-area research** — investigations that span multiple areas (Marketing + Operations + People + Tech + Data) before promotion to a single canonical home.
- **Engagement intelligence folders** — per-engagement intelligence-collection working space (e.g., [`2026-05-10-suez-webuy-procure-to-pay/`](2026-05-10-suez-webuy-procure-to-pay/)).
- **Substrate-audit quarterly folders** — `substrate-audit-YYYY-QN/` (e.g. [`substrate-audit-2026-Q2/`](substrate-audit-2026-Q2/)) per [`SOP-RESEARCH_SUBSTRATE_AUDIT_CADENCE_001.md`](../../references/hlk/v3.0/Admin/O5-1/Research/Methodology/canonicals/SOP-RESEARCH_SUBSTRATE_AUDIT_CADENCE_001.md) (I84 P6 paired SOP). Off-cycle event-trigger audits append a suffix (e.g. `substrate-audit-2026-Q3-off-cycle-cursor-ga`).
- **Knowledge-base drafts** — topic-fact-source drafts before promotion to `compliance/HLK_KM_TOPIC_FACT_SOURCE.md` (or its post-P4.5 federated home).
- **Ontology / taxonomy proposals** — classification-lattice candidates (P4 §11), 3-tier WIP topology refinements, future-OS-shape scenario investigations.
- **Methodology-pillar prototypes** — research-discipline candidates before promotion to `Research/Methodology/`.

## What does NOT live here

- **Initiative-scoped WIP** — lives at `docs/wip/planning/<NN-initiative-slug>/` (Tier 2; PMO-owned per blueprint §17).
- **Role-internal WIP** — lives at `<area>/<role>/wip/` (Tier 3; role-owned per blueprint §17).
- **Reference-only material** — third-party SOPs, regulatory texts, historical R&L corpus → `docs/references/hlk/Research & Logic/`.

## Promotion ladder (per blueprint §13)

1. **Investigate** — checkpoints, synthesis docs, evidence-matrix sketches under `<slug>/`.
2. **Propose** — candidate document at `<slug>/checkpoints/<NN>-promotion-proposal.md` declaring intent + target home + classification.
3. **Ratify** — inline-`AskQuestion` ratify gate per `akos-inline-ratification.mdc`; row added to `DECISION_REGISTER` (`D-PROMOTE-<slug>`).
4. **Promote** — canonical authored at target home; cross-references updated; WIP source moved to `<slug>/_promoted/<YYYY-MM-DD>-superseded-by-<canonical-path>/` audit trail.
