# AIC — Agent in Charge (RESERVED slot)

> This folder is **intentionally empty scaffolding** for the **AIC role-class**: any AI agent acting on behalf of a named role-owner with that role's full authority and accountability. Codified at I79 P5 cluster B (D-IH-79-J).

## Why this folder exists

The AKOS organisation chart hosts **three distinct AI governance positions** — listed in the parent vault index ([`v3.0/index.md`](../../../index.md) §"Vault Structure" lines 189–191):

1. **`Admin/AI/Susana Madeira/`** — the named **Holistika AI persona** with admin privileges.
2. **`Admin/AI/AIC/`** *(this folder)* — the **Agent-in-Charge role-class** that any role_owner may activate when delegating role-level execution to an AI agent.
3. The People-side jargon-free doctrine for both: [`Admin/O5-1/People/canonicals/HOLISTIKA_AGENTIC_DOCTRINE.md`](../../O5-1/People/canonicals/HOLISTIKA_AGENTIC_DOCTRINE.md) (authored at I79 P3a).

The folder is reserved because the AIC concept is **architecturally activated** but **not yet operationally instantiated**. The operational instantiation will land when:

- An I76 (or successor) candidate promotes the AIC pattern to canonical SOP coverage; **or**
- A role_owner explicitly designates an agent as their AIC for a defined scope and that designation is recorded as a canonical artifact.

When that lands, this `RESERVED.md` is replaced with the canonical AIC charter / register / SOP set. Until then, this marker exists so the next reader does not flag the folder as an orphan and delete it during housekeeping audits.

## What is **not** here

- This folder is **not** the place to author general agentic doctrine (that lives at [`HOLISTIKA_AGENTIC_DOCTRINE.md`](../../O5-1/People/canonicals/HOLISTIKA_AGENTIC_DOCTRINE.md)).
- This folder is **not** the place to author technical framework details (that lives at [`AGENTIC_FRAMEWORK_LANDSCAPE.md`](../../O5-1/Envoy Tech Lab/canonicals/AGENTIC_FRAMEWORK_LANDSCAPE.md)).
- This folder is **not** the place to bind ethical red-lines (those live at [`Admin/O5-1/People/Ethics/canonicals/ETHICAL_AGENTIC_BOUNDARIES.md`](../../O5-1/People/Ethics/canonicals/ETHICAL_AGENTIC_BOUNDARIES.md)).

## Relationship to the People manifesto

The People area owns the **discipline-of-disciplines** posture for agentic governance ([`HOLISTIKA_AGENTIC_DOCTRINE.md`](../../O5-1/People/canonicals/HOLISTIKA_AGENTIC_DOCTRINE.md) §1). When the AIC concept is operationalised, the AIC charter that lands here will inherit the People-side doctrine and reference (not redefine) the Tech Lab framework landscape.
