# Validation — discipline index

> **The discipline of _what's true_.** Validation is the truth-gate that runs after collection +
> diagnosis: source-reliability grading, cross-source corroboration, evidence-confidence scoring,
> and the gate decisions that license downstream promotion. Constitution:
> [`canonicals/VALIDATION_DISCIPLINE_CHARTER.md`](canonicals/VALIDATION_DISCIPLINE_CHARTER.md).
> Area spine: [`../canonicals/RESEARCH_LIFECYCLE_DOCTRINE.md`](../canonicals/RESEARCH_LIFECYCLE_DOCTRINE.md).

## Lifecycle ownership

**`++` primary at PROCESS** (the truth-gate) + the **feedback** loop; **+ confidence stamp** at
STORE; **+ claim-confidence gate** at SHARE; **+ reliability discipline** at PROTECT. Validation is
what earns a fact the right to be stored, shared, or promoted to canonical.

## Three validation moves + four gates (the structure)

Validation is a **new formalization** (cross-links existing compliance taxonomies; no legacy
technique folder). Its structure is the three moves + four gates from the charter:

| Move | Grades | Output |
|:---|:---|:---|
| Source-reliability grading | Single-source trust (HUMINT + OSINT) | A-F grade per [`confidence_levels.md`](../../Admin/O5-1/People/Compliance/canonicals/confidence_levels.md) |
| Cross-source corroboration | Multi-source convergence | Confidence score 1-5 per claim |
| Evidence-confidence scoring | Claim-level verdict | Confirmed / Probable / Possible / Doubtful / Improbable |

Four gates: pre-canonical-promotion · pre-customer-deliverable · pre-investor-deliverable ·
pre-engagement-acceptance (charter §3).
