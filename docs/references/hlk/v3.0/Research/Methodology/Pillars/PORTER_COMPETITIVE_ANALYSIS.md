# Porter Competitive Analysis — per-force viewpoint discipline

> **PROCESS-stage pillar.** Minted 2026-06-10 to operationalise operator Porter discipline absent from
> Methodology Pillars index (previously only a MADEIRA GTM task).

## 1. Operator discipline (binding)

Analyze the **same prong subject** from **each force's separate viewpoint**:

| Force | Viewpoint question |
|:---|:---|
| **1 — Supplier power** | Who supplies the capability this prong depends on? How much leverage do they have? |
| **2 — Buyer power** | Who can refuse, bypass, or commoditise this layer? |
| **3 — Substitutes** | What replaces this approach (one-off scripts, SaaS, manual SOP-only)? |
| **4 — New entrants** | What lowers the bar for new tools, adapters, or runbooks? |
| **∴ Competition** | **Synthesis of forces 1–4** — not a separate input pass |

> **Competitive rivalry is the result of analyzing the other four forces**, not a fifth independent
> lens. The synthesis row answers: "given supplier/buyer/substitute/entrant pressure, how intense
> is competitive constraint on this prong?"

## 2. Output shape

In each `prong-p*.md` synthesis file:

1. Four bullets (forces 1–4), each citing `SRC-*` where possible.
2. One **Competition (∴)** bullet integrating the four.
