# Diagnosis — discipline index

> **The discipline of _what's wrong_** — at the counterparty (engagement), the system (Holistika's
> own AKOS), or the methodology (per-pillar effectiveness). Diagnosis runs ON collected + validated
> intelligence and produces verdict + recommendation outputs. Constitution:
> [`canonicals/DIAGNOSIS_DISCIPLINE_CHARTER.md`](canonicals/DIAGNOSIS_DISCIPLINE_CHARTER.md).
> Area spine: [`../canonicals/RESEARCH_LIFECYCLE_DOCTRINE.md`](../canonicals/RESEARCH_LIFECYCLE_DOCTRINE.md).

## Lifecycle ownership

**`++` primary at PROCESS** — Diagnosis is the verdict layer (analysis/production in intelligence-
cycle terms). It consumes ACQUIRE output, sits beside Validation in PROCESS, and feeds SHARE
(the diagnostic report is a delivered product).

## Three diagnosis surfaces (no sub-folders; the surfaces are the structure)

Diagnosis is a **new** discipline (no legacy technique folder). Its structure is the three
diagnostic surfaces defined in the charter §2 — they are documented in prose + worked examples,
not technique sub-buckets:

| Surface | Object diagnosed | Output | Worked example |
|:---|:---|:---|:---|
| **Engagement diagnostic** | Counterparty's operational state | Per-engagement counterparty-baseline-reality report | SUEZ WeBuy procure-to-pay diagnostic |
| **System diagnostic** | Holistika's own AKOS (canonicals + pipelines + mirrors) | System-health audit | I70 P2.5 v3.0 vault audit |
| **Methodology diagnostic** | Holistika's own methodology coverage | Methodology gap analysis | I70 P2 7-gap synthesis |
