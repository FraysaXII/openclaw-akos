# Workflow: Analyze Repository

## Agent Sequence
1. **Architect** -- full codebase analysis
2. **Orchestrator** -- summary and recommendations

## Required Tools
- `list_directory`, `read_file`, `sequential_thinking`

## Steps
1. [ ] Scan top-level directory structure
2. [ ] Identify language, framework, and dependencies
3. [ ] Map module relationships and entry points
4. [ ] Assess test coverage and documentation completeness
5. [ ] Produce architecture summary with strengths and gaps

## Completion Criteria
- Architecture summary document produced
- At least 3 strengths and 3 gaps identified
- Recommendations prioritized by impact
